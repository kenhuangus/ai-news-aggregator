# Requirements: AI News Aggregator Open Source Release

**Defined:** 2026-01-24
**Core Value:** Make the pipeline work with standard API endpoints so anyone can run their own instance

## v1 Requirements

### Configuration Infrastructure

- [x] **CFG-01**: Create `config/providers.yaml` for LLM and image provider settings
- [x] **CFG-02**: Support environment variable interpolation in YAML (`${VAR}` syntax)
- [x] **CFG-03**: Fall back to current env vars (`ANTHROPIC_API_KEY`, etc.) if no config file exists
- [x] **CFG-04**: Validate config at startup with clear error messages
- [x] **CFG-05**: Create `config/providers.yaml.example` with templates for each provider mode

### LLM Provider Support

- [x] **LLM-01**: Support direct Anthropic API mode (x-api-key header auth)
- [x] **LLM-02**: Support OpenAI-compatible endpoint mode (Bearer auth for user proxies)
- [x] **LLM-03**: Allow configurable model name per mode
- [x] **LLM-04**: Preserve extended thinking functionality for Anthropic mode
- [x] **LLM-05**: Fail with clear error if extended thinking unavailable (ET is essential for quality)

### Image Provider Support

- [x] **IMG-01**: Support direct Google Gemini API mode (google-genai SDK, `generate_content()` API)
- [x] **IMG-02**: Support OpenAI-compatible endpoint mode (chat completions format for proxies)
- [x] **IMG-03**: Skip hero generation gracefully if no image provider configured
- [x] **IMG-04**: Log clear message when hero generation is skipped

### Prompt Abstraction

- [x] **PRM-01**: Extract all analyzer prompts to `config/prompts.yaml`
- [x] **PRM-02**: Extract orchestrator prompts (topic detection, executive summary)
- [x] **PRM-03**: Extract utility prompts (link enrichment, link following, ecosystem detection)
- [x] **PRM-04**: Support prompt customization without code changes
- [x] **PRM-05**: Create `config/prompts.yaml.example` with default prompts

### Documentation & Cleanup

- [x] **DOC-01**: Remove all RDSec hardcoded references from code
- [x] **DOC-02**: Rewrite README.md for external open-source audience
- [x] **DOC-03**: Update `.env.example` for public use (N/A - using providers.yaml instead)
- [x] **DOC-04**: Add Apache 2.0 LICENSE file
- [x] **DOC-05**: Convert `web/assets/logo.png` to `logo.webp` (logo.webp already exists)
- [x] **DOC-06**: Document provider modes and configuration in README
- [x] **DOC-07**: Document prompt customization in README

## Out of Scope

| Feature | Reason |
|---------|--------|
| Custom branding config | TrendAI branding stays (mascot, colors) — project identity |
| Alternative image providers (DALL-E) | Gemini-only for now, keeps scope manageable |
| Database backend | Keep file-based JSON output — simplicity |
| Streaming support | Not needed for batch pipeline |
| AWS Bedrock provider | Adds complexity, can add in v2 if requested |
| Provider abstraction library (LiteLLM dep) | Direct SDK calls preferred, keeps control |
